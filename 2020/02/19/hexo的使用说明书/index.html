搭建hexo博客过程总结

由于我使用的是windows10系统，所以先下载了git方便后续操作（使用cmd也可以）

step1 下载nodejs环境

  • hexo依赖于Nodejs环境，需要下载Nodejs,下载二进制压缩包即可。

  • 解压缩后将文件加目录载入环境变量。

  • 随后使用git bush查看node是否配置完成：node -v

step2 下载hexo

  • node配置完成后，开始下载hexo，但是由于国内镜像元（npm）速度很慢，所以先安装cnpm（淘宝镜像）：npm install -g cnpm –registry=http://registry.npm.taobao.org
  • 随后下载hexo：cmpm install -g hexo-cli

step3 创建自己的博客

  • 为博客创建一个目录blog，然后在目录中创建博客：hexo init
  • 启动博客：hexo s/sever
  • 创建新的文章：hexo n/new “title”,博文会被自动创建在souce/_posts目录下。可以使用vim或者编辑器直接写自己的博文。
  • 写完博客后，需要在网站上生成静态文件：hexo g/generate，然后再次运行博客，就可以在网站上看见自己更新的博客了。

step4 将博客推到github上

  • 在github上常见一个新的仓库，仓库名称一定要设置成github的username加上github.io，我的github的username是smallwhitezxw，所以我的仓库昵称是smallwhitezxw.github.io。

  • 回到blog目录下，需要先安装一个git的部署插件：*cnpm install –save hexo-deployer-git

  • 随后需要我们配置_config.yml文件：

    配置_config.yml文件：

    ​ deploy:
    ​ type: ‘git’
    ​ repo: https://github.com/smallwhitezxw/smallwhitezxw.github.io.git
    ​ branch: master

    其中repo是仓库的地址，注意在每一个:的后面都需要加一个空格，否则无效！！！

  • 部署到远端：hexo d

    坑！！：由于我们是使用git进行操作，所以在往远端推之前，必须配置git的user.name和user.email：

    • 如果你的Git还没有初始化，可以通过这个指令：

      git config –global user.name “输入你的用户名”

      git config –global user.email “输入你的邮箱”

    • 如果你的git已经初始化并且需要修改：

      git config –global –replace-all user.email “我的邮箱”

      git config –global –replace-all user.name “用户名”

  • 部署完成后，使用smallwhitezxw.github.io即可访问自己的博客了！！！
