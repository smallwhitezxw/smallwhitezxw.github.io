python模块与包

1.模块

模块的分类

  • 内建模块
  • 标准库模块
  • 第三方模块
  • 用户自己编写的模块

模块的导入语句种类：

  1. import 语句
  2. from import 语句
  3. from import *

模块的搜索顺序：

  1. 搜索程序运行时的路径（即当前路径）

  2. sys.path提供的路径

    • sys.path是一个列表，里面放的都是模块的搜索路径，

      1. 可以将模块的py包移动到sys.path里存在的路径下

      2. 可以用sys.path.append()函数添加路径。

  3. 搜索内建模块

模块的加载过程

  1. 在模块导入时，模块的所有语句都会执行。

  2. 如果一个模块已经导入，则再次导入时不会重新执行所有模块内的语句，但是可以用imp.reload()函数重新加载。

  3. 模块导入和执行的过程：（）

    1. 先搜索相关的路径，找到模块名.py
    2. 判断是否有此模块对应的.pyc文件。如果.pyc文件比.py文件新，则直接加载.pyc文件。
    3. 否则，用模块.py文件生成.pyc，并加载执行。

模块的各种属性

  • 模块的__doc__属性：此属性用于绑定模块的文档字符串
    • 模块内第一个没有赋值给任何变量的字符成为模块的文档字符串
  • 模块的__file__属性：
    • 此属性用于记录模块对应的文件路径
  • 模块的__name__属性：
    • 用来记录模块自身的名字。
    • 作用：
      1. 记录模块名
      2. 用来判断是否为主模块（当模块为主模块（也就是第一个运行的模块）运行时，_name_绑定’__main\‘，模块不是主模块，而是被其他模块导入时，__name__绑定模块名）
  • 模块的__all__属性（仅作用与from import *语句）：
    • 在模块前加入_all_=[]列表，在列表内添加模块内的变量名，则该模块在被以from import * 方式调用时，仅仅会调用__all__列表内的变量！4
  • 模块的隐藏属性：
    • 模块中以’_’开头的属性，在from xxx import * 导包时，将不被导入，通常称这些属性（变量）为隐藏属性

dir函数：返回当前模块有哪些变量

random模块：

2.包

定义：包是将模块以文件夹的组织形式进行分组管理的方法。

作用：

1. 将一系列模块进行分类管理，有利于防止命名冲突。
  2. 可以在需要时加载一个或部分模块则不是全部模块。

_init_.py文件

  • _init_.py文件是常规包内必须存在的文件，_init_.py文件会在包加载时自动调用。

作用：

1. 编写此包的内容
 2. 在内部填写文档字符串
 3. 在\__init\__.py文件内可以加载此包所依赖的一些其他模块包的导入。

_init_.py内的__all__列表：

  • 作用：用来记录此包中有哪些子明或模块再用from import *语句时被导入。
  • 说明：__all__列表只对from import *语句起作用

包的导入：

同模块导入规则：

  1. import语句
    • import 包名 [as 自定义名]
    • import 包名.模块名 [as 自定义名]
    • import 包名.子包名.模块名 [as 自定义名]
  2. from import语句
    • from 包名 import 模块名 [as 自定义名]
    • from 包名.模块名 import 模块名 [as 自定义名]
    • from 包名.子包名.模块名 [as 自定义名]
  3. from import * 语句
    • from 包名 import *
    • from 包名.模块名 import *

导入包的索引路径顺序：

  1. 搜索程序的当前路径
  2. sys.path提供的路径（同模块相同可以更改）

包的相对导入：

  • 包的相对导入是指包内模块的相互导入
  • 语法：
    • from 相对路径包或模块 import 属性或模块名
    • from 相对路径包或模块 import *
  • 相对路径：
    • .代表当前目录
    • ..代表上一级目录
    • ...代表上二级目录
    • ....以此类推
    • 注：相对导入不能超出包的外部！
